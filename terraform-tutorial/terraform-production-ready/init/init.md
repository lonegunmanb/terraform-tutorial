# 生产就绪代码：从大泥球到模块化

本实验从一个"一锅炖"的单体 Terraform 配置出发，一步步重构出一套清晰、可复用、可测试的模块化基础设施。

## 我们要构建什么

一个**配置变更通知系统**，包含三类资源：

- S3 存储桶：存储应用配置文件（开启版本控制）
- SQS 队列：配置变更事件通知（含死信队列）
- DynamoDB 表：变更操作审计日志
- IAM 策略：应用访问权限

这套系统麻雀虽小，五脏俱全——涵盖了存储、消息、数据、权限四个常见基础设施层。

## 实验环境

- Terraform 1.14+
- AWS Provider 6.x
- MiniStack（模拟 AWS，支持 S3 / SQS / DynamoDB / IAM）
- 工作目录：/root/workspace/

你已经深入理解了 Terraform 状态管理的核心概念和操作：

## 核心概念回顾

- **状态文件是 Terraform 的"记忆"** — 记录了代码与真实资源的映射关系，信息远比代码详细
- **管理边界** — Terraform 只追踪状态文件中记录的资源，不会干扰外部管理的资源
- **三方对比** — `terraform plan` 对比代码、状态文件、真实环境三者，找出差异
- **漂移检测** — 当真实环境被外部修改时，Terraform 能自动发现并修复漂移
- **状态文件的必要性** — 没有状态文件，Terraform 无法知道被删除的代码曾对应一个真实资源，也就无法生成销毁计划

## 命令速查

| 命令 | 作用 |
|------|------|
| `terraform state list` | 列出所有被 Terraform 追踪的资源 |
| `terraform state show` | 查看资源在状态文件中的完整属性 |

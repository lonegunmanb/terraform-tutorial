# Checkov 实战

通过本节实验，你将体验 Checkov 如何对 Terraform 代码进行安全合规扫描，发现不安全配置并修复它们。

## 实验场景

工作目录中有一个故意包含多种安全问题的 Terraform 项目：

- S3 存储桶未开启版本控制
- S3 存储桶未开启服务端加密
- S3 存储桶未开启访问日志
- S3 存储桶允许公共访问
- Provider 中硬编码了凭证（虽然是测试用的假凭证，但 Checkov 仍会标记）

你将使用 Checkov 发现这些安全问题，理解每条规则的含义，然后逐步修复。

## 学习内容

| 内容 | 说明 |
|------|------|
| 基础扫描 | 使用 Checkov 扫描代码并理解输出格式 |
| 规则理解 | 理解 CKV_AWS_* 规则的含义和修复方法 |
| 问题修复 | 根据扫描报告修复安全问题 |
| 规则管理 | 跳过特定规则、只运行指定规则 |
| 自定义策略 | 编写 YAML 自定义策略检查 |

点击右侧箭头开始实验。

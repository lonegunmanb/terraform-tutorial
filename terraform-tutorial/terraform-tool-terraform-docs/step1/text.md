# 生成文档、配置注入与自定义模板

## 验证安装

先进入工作目录（所有 .tf 文件在这里）：

```
cd /root/workspace
ls *.tf
```

确认看到 main.tf、variables.tf、outputs.tf 三个文件后，验证 terraform-docs 已安装：

```
terraform-docs version
```

## 查看模块代码

```
cat main.tf
cat variables.tf
cat outputs.tf
```

这是一个 S3 静态网站托管模块，包含 7 个输入变量、4 个输出值。

## 生成 Markdown 文档

最基本的用法——在终端打印 Markdown 格式文档：

```
terraform-docs markdown .
```

输出包含 Requirements、Providers、Resources、Inputs、Outputs 等区块。

换成表格格式——更紧凑：

```
terraform-docs markdown table .
```

注意 Inputs 表格中包含了每个变量的 Name、Description、Type、Default 和 Required 信息——全部从 variables.tf 自动提取。

## 其他输出格式

彩色简洁格式：

```
terraform-docs pretty .
```

JSON 格式（适合程序解析）：

```
terraform-docs json . | head -30
```

生成 tfvars 模板——包含所有变量的默认值，方便用户快速创建 terraform.tfvars：

```
terraform-docs tfvars hcl .
```

## 注入到 README.md

创建一个带标记的 README.md：

```
cat > README.md <<'EOF'
# S3 Static Website Module

This module creates an S3 bucket configured for static website hosting.

## Usage

```hcl
module "website" {
  source      = "./path/to/module"
  bucket_name = "my-website"
}
```

## Documentation

The following documentation is auto-generated by terraform-docs.

<!-- BEGIN_TF_DOCS -->
<!-- END_TF_DOCS -->

## License

MIT
EOF
```

运行注入命令：

```
terraform-docs markdown table --output-file README.md --output-mode inject .
```

查看结果：

```
cat README.md
```

生成的文档被插入到 BEGIN_TF_DOCS 和 END_TF_DOCS 标记之间，标记之外的自定义内容（Usage、License 等）完好保留。

## 使用配置文件

创建 .terraform-docs.yml 配置文件：

```
cat > .terraform-docs.yml <<'EOF'
formatter: markdown table

output:
  file: README.md
  mode: inject
  template: |-
    <!-- BEGIN_TF_DOCS -->
    {{ .Content }}
    <!-- END_TF_DOCS -->

sort:
  enabled: true
  by: name

settings:
  default: true
  required: true
  type: true
  description: true
EOF
```

有了配置文件后，只需运行：

```
terraform-docs .
```

不需要任何命令行参数。查看更新后的 README：

```
cat README.md
```

## 自定义内容模板

用 content 字段完全控制文档结构：

```
cat > .terraform-docs.yml <<'EOF'
formatter: markdown table

output:
  file: README.md
  mode: inject
  template: |-
    <!-- BEGIN_TF_DOCS -->
    {{ .Content }}
    <!-- END_TF_DOCS -->

content: |-
  {{ .Header }}

  ## Requirements

  {{ .Requirements }}

  ## Inputs

  {{ .Inputs }}

  ## Outputs

  {{ .Outputs }}

  ## Resources

  {{ .Resources }}

sort:
  enabled: true
  by: name
EOF
```

重新生成：

```
terraform-docs .
cat README.md
```

现在文档的区块顺序和结构完全按你定义的模板排列。

## 隐藏特定区块

如果不想在文档中显示 Providers 和 Modules 信息：

```
cat > .terraform-docs.yml <<'EOF'
formatter: markdown table

sections:
  hide:
    - providers
    - modules

output:
  file: README.md
  mode: inject
  template: |-
    <!-- BEGIN_TF_DOCS -->
    {{ .Content }}
    <!-- END_TF_DOCS -->

sort:
  enabled: true
  by: name
EOF
```

```
terraform-docs .
cat README.md
```

Providers 区块不再出现在文档中。

## 幂等性验证

再次运行 terraform-docs，确认不会产生多余变更：

```
cp README.md README-before.md
terraform-docs .
diff README.md README-before.md && echo "文档已是最新" || echo "文档有变更"
rm README-before.md
```

这个检查逻辑可以直接用在 CI 中——如果 diff 有输出，说明开发者忘了更新文档。

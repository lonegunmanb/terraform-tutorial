你已经掌握了 Terraform 模块化的核心知识：

| 概念 | 说明 |
|------|------|
| 模块结构 | `variables.tf` + `main.tf` + `outputs.tf` |
| 调用模块 | `module "name" { source = "..." }` |
| 传参 | 通过模块变量传入配置 |
| 输出引用 | `module.<name>.<output>` |
| 复用 | 同一个模块可被多次调用，传入不同参数 |

## 模块设计原则

- **单一职责**：一个模块只做一件事
- **合理暴露变量**：让调用方能定制，但不要暴露过多实现细节
- **为变量写 description**：让使用者不用看代码就知道该填什么
- **使用 validation**：在模块层面约束输入，提前报错
